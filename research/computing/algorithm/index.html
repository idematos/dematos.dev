An algorithm is a sequence of instructions to perform an action.

There are several ways to perform an action. Each algorithm requires a different amount of energy to execute. For the purposes of this context, we can call this amount energy complexity.

Simply put, the energy complexity of an algorithm depends on the number of operations performed, the type of operations, and the hardware's efficiency. Algorithms with lower time complexity generally consume less energy because they perform fewer operations. For example, a merge sort (O(n log n)) is more energy-efficient than a bubble sort (O(n²)) for large datasets.

Some ways to optimise algorithms and data structures

  • Replacing, when possible, algorithms with a version that achieves the same results with a lower time complexity. For example, when processing graphs, Dijkstra's algorithm (O(V²)) can be replaced by the Fibonacci heap implementation (O(V²)) can be replaced for the Fibonacci heap implementation (O(V log V + E)).

  • Using approximation algorithms when exact solutions are computationally expensive. For example, approximate nearest-neighbour algorithms significantly reduce energy costs in high-dimensional data searches.

  • Reducing precision in computation, particularly in machine learning tasks. For instance, using 16-bit floating-point operations instead of 32-bit reduces energy costs with minimal accuracy loss.

  • Replacing linked lists with dynamic arrays for random access operations.

  • Using compact data structures, such as succinct trees or bloom filters.
