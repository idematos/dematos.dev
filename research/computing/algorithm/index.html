An algorithm is a sequence of instructions to perform an action.

There are several ways to perform an action. Each of these ways is a different algorithm, and each algorithm requires a different amount of energy to perform. This amount is called energy complexity.

The energy complexity of an algorithm depends on the number of operations performed, the type of these operations and the efficiency of the hardware on which it is being executed. Algorithms with lower time complexity generally consume less energy because they perform fewer operations. For example, a merge sort (O(n log n)) is more energy-efficient than a bubble sort (O(n²)) for large datasets.

Some ways to reduce energy complexity in algorithms and data structures

  • Using algorithms with lower time complexity whenever possible. For example, when processing graphs, Dijkstra's algorithm (O(V²)) can be replaced by the Fibonacci heap implementation (O(E + V log V)).

  • Using approximation algorithms when exact solutions are computationally expensive. For example, approximate nearest-neighbour algorithms significantly reduce energy complexity in high-dimensional data searches.

  • Reducing precision in machine learning tasks. For instance, using 16-bit floating-point operations instead of 32-bit reduces energy complexity with minimal accuracy loss.

  • Replacing linked lists with dynamic arrays for random access operations.

  • Using compact data structures, such as succinct trees or bloom filters.
